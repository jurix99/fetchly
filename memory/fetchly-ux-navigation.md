## Carte mode (M├®moire, v0.0.12)
Local graph exploration, ALWAYS centred on one content (never a whole-library constellation; no graph lib, no physics ŌĆö hand-rolled deterministic radial layout).
- **Backend** [indexer.py](app/indexer.py): `content_map(content_id, depth=1|2)` composes cached `related()` payloads ŌåÆ `{center_id, depth, nodes[], edges[]}`. Nodes `{content_id,title,thumbnail,duration,channel,score_to_center,ring}` (ring 0/1/2). Edges `{a,b,score,pair}` incl. neighbourŌåöneighbour above `_REL_MIN_SCORE`, deduped undirected. Caps `_MAP_MAX_D1=12`, `_MAP_MAX_TOTAL=25`. `map_start()` = last-opened-with-links else most-connected (`db.map_start_candidates`). **`related()` now caches full top-`_REL_CACHE_MAX=25` and slices to `limit` on read** (was caching by content_id only, ignoring limit ŌĆö the map needs >5). Routes: `GET /api/library/{id}/map?depth=`, `GET /api/library/map/start` (declared before `{content_id}/related`; distinct segment counts so no shadowing).
- **Frontend** [memory-map.tsx](frontend/components/memory-map.tsx): radial layout (center + 2 rings, size ŌłØ score, px positions from ResizeObserver), SVG edges (thickness ŌłØ score, `var(--color-primary/border)`), hover/tap edge tooltip (IciŌåöL├Ā-bas passage pair, timestamps seek), node ŌåÆ `Sheet` drawer (knowledge card + Ouvrir/Centrer + links list = accessible no-hover path), breadcrumb trail, depth 1/2 toggle (mobile forced 1), keyboard arrows between/along rings. Wired in [library-view.tsx](frontend/components/views/library-view.tsx) as 3rd mode (grid/list/**map**, Carte no longer disabled); `mapCenter` prop from app-shell `openMap()`; "Ouvrir la carte" button in content-detail RelatedSection.
